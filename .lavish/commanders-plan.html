Commanders & Command Powers

Four factions, twelve generals, each with a permanent Doctrine and one chargeable Command Power. This plan maps every doctrine onto the systems the sim already has, proposes a single hook layer so commander rules never leak into resolvers, and cuts the work into four milestones that each end in something playable.

1 · Why this is cheaper than it looks

Every subsystem the twelve doctrines touch already exists in core/ and is already unit-tested. The feature is therefore one new economy (charge) + one new command + a modifier layer, not new mechanics. Verified against the code:

Her power "Vanish: units in Woods can only be revealed from an adjacent tile during fog" was already the base rule for everyone: vision.gd:43 hides any unit in woods from non-adjacent viewers, so that clause was a no-op as written. Decided (D4): Vanish is reworked to true invisibility — during the power her woods units are hidden even from adjacent tiles, revealed only when an enemy tries to enter their cell; the +40% first strike from woods stays.

3 · Architecture — one hook layer, zero commander ifs in resolvers

The question the design must answer: where do commander rules live so that twelve generals don't smear special cases across every resolver? Answer: a single CommanderType Resource base class whose hook methods return neutral defaults. Resolvers call the hooks unconditionally; a match without commanders uses the neutral instance and behaves exactly as today. Each general is a tiny subclass in core/commanders/ overriding two or three hooks, with its balance numbers as @exports on a .tres in data/commanders/ — behaviour in small typed classes, numbers in data, exactly like the rest of the repo.

The hook API

All hooks live on CommanderType, take sim state only (no Nodes), and check power_active themselves so a power is just "the hook returns a bigger number while active". Percent hooks return integer percentage points — no float creep.

HookNeutral defaultWave-1 usersCall site
attack_bonus(state, unit, target, is_counter) -> int 0 Alina, Viktor, Rhea, Tomas + every offensive power combat_resolver · _damage_pct
defense_bonus(state, unit, attacker) -> int 0 Rhea (−10 on indirects), Coordinated Push combat_resolver · _damage_pct
star_bonus(state, defender) / star_pierce(state, attacker) -> int 0 Armoured Breakthrough (pierce 1); Sable passive later combat_resolver · _damage_pct
luck_min(state) / luck_max(state) -> int 0 / 9 — (Lyra & Perfect Solution, wave 2) combat_resolver · resolve

terrain_cost(unit, terrain, base) -> int base Nia (woods/mountains −1, min 1, foot & boot) movement_resolver + game_state.advance_unit (fuel)

Damage formula, updated

stars  = clampi(terrain_stars + def_co.star_bonus − att_co.star_pierce, 0, 5)

att    = 100 + att_co.attack_bonus(unit, target, is_counter)

def    = 100 + def_co.defense_bonus(unit, attacker)

raw    = base × att/100 × att_hp/10 × (1 − 0.1 × stars × def_hp/10) × (200 − def)/100

damage = maxi(0, roundi(raw))   ·   resolve adds luck = rng.randi_range(luck_min, luck_max)

The (200 − def)/100 term is the classic Advance Wars defence shape: +10 defence → ×0.9 damage, −10 → ×1.1. Multiplier order and the single final round are fixed here so every general is balance-tested against the same math. Both forecast() and resolve() go through _damage_pct, so the damage preview reflects doctrines automatically.

4 · Charge meter & Command Powers

Value-weighted charge — integer only

Your formula, kept, but in integer charge points so saves and replays stay exact (internal HP is already 0–100, i.e. a percentage):

charge += unit.type.cost × hp_lost / 100

Halving a 7 000-cost Tank ⇒ 7000 × 50 / 100 = 3 500 points — your "3.5 charge" at ×1000 scale. Powers cost 8 000–14 000 points. Accrues in CombatResolver.resolve() for both the attack and the counter, capped at the power's cost. Decided (D2) — asymmetric split: the side losing HP charges at 100%, the side dealing it at 50% — the classic comeback pressure that stops one player snowballing meter and match at once.

PowerCommand — just another command

validate: it's your turn, charge ≥ power_cost, no power already active this turn. apply: deduct the full cost, set power_active, run on_power_activated() for one-shot effects (Open the Depots' refill + heal). It enters the same command log as everything else, so saves, replays, and the AI get it for free.

Duration: an enum on CommanderType. OWNER_TURN powers deactivate in EndTurnCommand; ROUND powers (only Hold the Line so far) deactivate at the owner's next begin_turn. Two clear expiry points, no ambiguous timing, easy for the AI to evaluate.

5 · Wave 1, general by general

Alina Ward C1

Hooks: attack_bonus scans the four neighbours for a friendly with a different move_class (all four classes already exist on UnitType). Power adds flat attack/defence/move bonuses while active.

Tests: bonus on/off by adjacency; Infantry+Mech (foot vs boot) counts as different; power raises movement in MovementResolver.reachable and expires on end turn.

Viktor Draeg C1

Hooks: attack_bonus keyed on unit id (tank/md_tank +15, infantry/mech −10). Power: move_bonus +1 for treads, star_pierce 1.

Tests: tank damage up, infantry damage down, recon unchanged; pierced star vs a city (3★→2★) matches hand-computed damage; stars clamp at 0.

Gideon Holt C2

Hooks: supply_range() → 2 (turn-start supply and the Supply action both switch from hardcoded adjacency to the hook); repair_cost_pct() → 80. Power is pure on_power_activated: resupply every unit, +10 internal HP.

Tests: supply at distance 2 but not 3; repair charges 80% of the proportional cost; power heals 1 displayed HP and refills a drained tank.

Tomas Reed C2

Hooks: attack_bonus (foot +15, vehicles −10); capture_bonus_pct +20 ⇒ capture chip becomes displayed_hp × (100 + bonus) / 100 rounded down. Power: +100 (double) and move_bonus +1 for foot units.

Tests: 10-HP infantry chips 12 points not 10; uprising captures a city in one turn from 20 points with 10-HP infantry (20 chip); vehicle attack penalty visible in forecast.

Rhea Sol C3

Hooks: attack_bonus +10 / defense_bonus −10 for min_range > 1 units. Power: range_bonus +1 and attack +20. Requires the shared attack_range() helper (R3) so command validation, AI targeting, and the UI range overlay all agree.

Tests: artillery hits at range 4 only while the power is active; AI finds the extended-range shot; counter rules unchanged (indirects still never counter).

Nia Rowan C3

Hooks: terrain_cost (woods/mountain −1, min 1, foot & boot) — called from both the Dijkstra step and advance_unit's fuel spend so fuel matches the path cost the player saw. Power: move_bonus and vision_bonus +1, sees_into_woods true for foot + recon.

Tests: infantry reaches one hex deeper through woods; fuel spent equals discounted cost; under fog, Ghost March reveals a woods tile at range that stays hidden for a neutral commander.

    Mara, Cass, Sable (reworked), Lyra, Orin, Cassian. Adds the three remaining hook needs: the is_counter flag (Mara), luck-range hooks (Lyra), and enemy-targeting power effects (Signal Jam). Then a balance pass over data/commanders/ — numbers only, no code.

7 · The AI

Doctrine awareness: free

The AI already scores attacks through CombatResolver.forecast() (ai_controller.gd:102). Since doctrines land inside _damage_pct, the AI's damage estimates, counter-risk math, and target choice adjust to any commander automatically — no AI change needed for passives.

Power firing: deliberately simple v1

In plan_next_command, before per-unit planning: if the meter is full and at least one enemy is inside any unit's threat range, issue PowerCommand. Deterministic, credible, and correct-by-construction for OWNER_TURN powers since they only help this turn's actions. C4 stretch: per-commander gating (Uprising waits for a reachable capture; Hold the Line fires on the defensive), still rule-based, no lookahead.

8 · Presentation layer

Main menu

One commander dropdown per side next to the existing map/fog options ("No Commander" default keeps the current experience). Selection rides in MatchConfig.commanders exactly like ai_teams and fog_enabled do today; a --co CLI flag keeps headless demos working.

Battle HUD

Charge meter (progress bar, charge / power_cost) beside the funds display, with a Power button that enables when full — it issues PowerCommand like any other input. Active power = tinted banner strip + meter glow. EventBus gains power_activated(team, commander) for a short banner animation and an Sfx sting.

Faction identity (later)

Faction palettes (navy/gold, red/black, green/tan, white/violet) can start as unit tint variations on the existing atlas rows; portraits and faction-skinned UI are content work, deliberately outside these milestones. Any sourced art gets tracked in assets/LICENSES.md.

9 · Risks & failure modes

#RiskMitigation
R1 Silent balance drift — reordering multipliers or rounding twice changes damage everywhere. The formula in §3 is normative: one multiplier chain, one final roundi. A golden-value test pins a matrix of hand-computed damages for neutral + each wave-1 commander.
R2 Determinism breaks — a power effect or charge calc using floats or global RNG desyncs replays. Charge is integer math; luck stays on state.rng; PowerCommand lives in the command log. Existing seeded-replay tests extended to a match with powers.
R3 Range duplication — min/max range is checked in attack_command.gd:36, ai_controller.gd:98, and the counter check. Adding Rhea's bonus to only some of them makes the UI, AI and rules disagree. C3 introduces one shared attack_range(state, unit) helper first, migrates all three call sites with the hook returning 0, then ships Rhea.
R4 Save compatibility — v1 saves have no commander block. Missing keys default to the neutral commander with 0 charge; a v1-fixture load test guards it.
R5 Charge farming — feeding cheap units to charge the meter. The value-weighted formula already caps an Infantry sacrifice at 1 000 points vs an 8 000+ power. The asymmetric split (D2) further favours the side actually taking valuable losses. Watch it in playtesting, tune costs in data.
R6 Scope creep — the plan's named top risk. Twelve generals, portraits, super powers, per-CO music… Wave 1 is six generals with zero new art. Super Powers explicitly out of scope. Faction skins deferred. Every milestone ships a playable, testable increment.
