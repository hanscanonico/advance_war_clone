Command Power Quotes — the general speaks

Today a Command Power announces itself with a fanfare and a stat card: portrait, power name, rules text. Correct, and completely voiceless. This plan gives each of the twelve generals a short in-character line spoken on the activation banner — the genre's signature beat, the moment the player learns who they're fighting, not just what modifier just turned on. It is a string on the commander's .tres and one new label on an existing banner: no line of core/ or ai/ ever reads a quote, no RNG is touched, and the same seam that announces a player's power already announces the AI's.

Meridian — “Advance as one.” Iron — “Steel does not step back.” Verdant — “We were never here.” Aurora — “Variance eliminated.”
1

The seams already exist

Almost everything this feature needs was built for the Commanders plan and is sitting idle. The work is one field, one label, and twelve short scripts:

One announce seam, both actors

battle.gd:539 — _announce_power(fired) plays the fanfare, shows the banner, and emits the event. The player's Fire button reaches it through _fire_command_power (battle.gd:526) and the AI reaches the same function from battle_ai_runner.gd:134. A quote added here covers both sides by construction — no player/AI split to keep in sync, the exact failure mode the hook-layer soak test exists to catch elsewhere.

The command already carries the payload

core/commands/power_command.gd:14 — PowerCommand.apply() records team and commander explicitly for the presentation layer, and tests/unit/test_power_command.gd:83 already asserts it. autoload/event_bus.gd:13 even declares power_activated(team, commander) — emitted today, consumed by nobody. The plumbing predates the feature.

The banner card is already on screen

scenes/ui/commander_power_banner.gd:74 — CommanderPowerBanner.bind(commander) already lays out the portrait (CommanderVisuals.portrait_for), the faction tint (theme_for), the “NAME · COMMAND POWER” eyebrow, the power name and the rules text. It already holds open under the capturing flag (battle_animator.gd:253) and already times itself through Settings.speed.power_banner_seconds() (game_speed.gd:174 — 1.1 s, 0.5 s under Instant). The quote is one more label on a card that has already solved tinting, portraits, screenshots and pacing.

Presentation keys on core resources have a precedent

The battle-animations plan's D5: UnitType.battle_style is a presentation key living on a core-read resource, exactly like atlas_col. core/commander_type.gd already carries display_name, faction, doctrine_text and power_text (lines 47–54) — strings the sim never reads. A power_quotes field is the same move, and the scenario driver's _stage_power_banner() (battle_scenario_driver.gd:484) already photographs the surface it will appear on.

What each general says

Three lines per general, rotated in order across a match (§3 D2). A line is a character beat, not rules text — the card below it still states what the power does. House style: short enough to read at a 1.5 s hold (the completeness test enforces a hard length cap), voiced to the doctrine, never naming a game term the card already shows. These are starting scripts — the writing pass in PQ1 owns the final words the way a balance pass owns numbers.

Where the line sits on the card

Alina Ward · Command Power

“All units, advance as one. No one fights alone today.”

Coordinated Push

All units +1 movement, +10% attack, +10% defence this turn.

The line reads as speech, not caption: the general's portrait sits beside it, face left, words right — the dialogue-box arrangement the genre made canonical. This is a requirement, not an inherited nicety: a quote never appears without the portrait on screen saying it, and CommanderVisuals.portrait_for already guarantees a face for every id (it falls back to the neutral bust). The quote is the card's headline — large, italic, in the general's voice — with the power name and rules text stepping down a size beneath it. The character speaks first; the rules clarify after. Everything else about the card (faction tint, border, hold, centering) is untouched. An empty power_quotes hides the label and renders today's card pixel for pixel, which is the regression story in one sentence.

3

Decisions

D1 · Where does quote text live? locked

As @export_multiline var power_quotes: Array[String] on core/commander_type.gd, defaulting to empty, with the words in each general's .tres under data/commanders/. This is the battle_style pattern: a presentation key on a core-read resource that nothing in core/ or ai/ ever reads — and it sits beside doctrine_text / power_text, which are already exactly such strings. Writing a general's voice becomes editing their data file, the same motion as balancing their numbers. Rejected: a parallel presentation resource keyed by commander id — a thirteenth file family to keep in sync, solving a layering problem this repo has already decided doesn't exist for display strings.

D2 · One line or many — and who picks? locked

Three lines each, rotated in order by that side's activation count — no RNG anywhere. The banner keeps a per-team counter and shows quotes[count % size]. Deterministic by construction: two runs of the same replay speak the same lines, the scenario gallery's power-banner frame is always activation #1 and so always the same words, and the sim's seeded RNG is never touched. The counter is presentation state that resets on scene load — resuming a save restarts the rotation, which costs nothing anyone can measure. Rejected: random pick (this repo verifies presentation by diffing frames; a die roll in the theatre is a diff that lies) and a single line (the charge-reset rule means a long match hears the power four or five times — one sentence five times is how charm becomes wallpaper).

D3 · Where does it render? locked

Inside the existing CommanderPowerBanner, as the card's new headline — not a sibling speech bubble, not a new cut-in. bind() grows a team parameter (for the rotation counter) and one label; show_power_banner passes fired.team through from _announce_power. The layout obligation that comes with the choice: the quote renders beside the portrait, as the portrait's speech — the face must be on screen whenever the line is, which the card satisfies by construction since bind() always draws the bust. Everything hard about the surface — portrait, faction tint, mirror-match colours (SideIdentity's borrowed-classic rule), the capturing hold, Instant's clamp — is inherited rather than re-solved. Rejected: a full-screen letterbox cut-in in the style of the battle animation. The combat cut-in earns its ceremony with art it already has; a power quote has a portrait and a sentence, and a banner is the honest size for that. If portrait art ever grows animation frames, the card is still where they'd land.

D4 · Does the quote block anything? locked

No. The power banner is fire-and-forget today (_announce_power is not awaited by either caller) and stays that way — the AI keeps acting under it, the player keeps their cursor. The one number that moves: POWER_BANNER_SECONDS 1.1 → 1.5, because the card now opens with a sentence and 1.1 s was sized for a glance at a title. Instant's 0.5 s clamp is untouched — under Instant the quote is a flicker you can choose to read, in exactly the way the game-speed plan treats all banners: information, not theatre. The retune is a one-constant edit in the one place pacing lives (game_speed.gd:44), settled by playing, like GS3's outstanding pass.

D5 · Is there a voice? locked

Text only; the existing &"fanfare" sting stays the sound of a power. The door to per-commander VO is already open — Sfx silently skips missing streams (sfx.gd:47), and the BattleStyle "a style names its own sound" precedent shows how data-named audio ships — but recording twelve voices is content work, and a text line that ships this week beats a voice line that never does. If VO ever lands, the quote field is the script it reads from.

4

Milestones — each one shippable

PQ1 · The banner speaks size S shipped

The field, the twelve scripts, and the card's new headline — the whole player-facing feature in one milestone.

  • core/commander_type.gd — power_quotes beside power_text, empty default. Inert to the sim by construction; the neutral commander and any quote-less .tres render today's card unchanged.
  • All twelve data/commanders/*.tres get their three lines (§2's scripts, given a writing pass as they land — final words are an editing decision, not a code one).
  • commander_power_banner.gd — bind(commander, team) adds the quote headline (hidden when empty) and the per-team rotation counter; battle_animator.gd:247 passes fired.team through show_power_banner.
  • The quote lays out beside the portrait (face left, words right) so the line reads as spoken by the general — a quote never renders without the bust, which CommanderVisuals.portrait_for's neutral fallback makes unconditional.
  • POWER_BANNER_SECONDS → 1.5 in game_speed.gd; Instant clamp untouched; capturing hold untouched.
  • New tests/unit/test_commander_quotes.gd — every commander .tres with a power ships ≥ 1 non-empty quote, and every quote respects the length cap (a test constant, the editorial ruler). Node-free resource loading, so it belongs in GUT.
  • Proof: fire a power as the player and watch the AI's next activation speak through the same seam; three activations rotate three lines; make test green; make screenshot byte-stable (the capture path never stages a power).

PQ2 · The voice spreads to the card family size XS shipped

The quote exists everywhere the commander's face does, and the galleries acknowledge the new card.

  • The commander info sheet (scenes/ui/commander_info_sheet.gd) shows the signature line — quotes[0], the same one the gallery's power-banner frame shows — so the select screen introduces the character the battle will deliver.
  • Re-bless the scenario gallery's power_banner stage capture (battle_scenario_driver.gd:484): one expected, one-time frame change, deterministic thereafter because the stage is always activation #1.
  • README / docs note under the commanders section: quotes are data, where they live, the length cap, the rotation rule.
5

Risks

R1 · The card becomes a poster

Portrait, eyebrow, quote, power name, rules text — five things on one banner is a real crowding risk. Mitigation is editorial, enforced mechanically: the length cap lives in the completeness test, the quote gets exactly one line of visual hierarchy (the headline), and nothing else on the card grows. If a line can't fit the cap, the line is wrong, not the cap.

R2 · Charm becomes wallpaper

The charge-reset rule means a long match hears several activations; a line that landed on Day 3 grates by Day 12. Three rotated lines is the floor that makes repetition survivable, not a guarantee of delight — if playtest still tires of them, the fix is more lines in the .tres (data, no code), not a cleverer picker.

R3 · Determinism, saves, replays

Structurally safe: the only core/ change is an @export string array nobody in the sim reads, the command log and save format are untouched (save_codec.gd already round-trips the commander id the quote derives from), and rotation counts activations instead of rolling dice. Same seed, same commands, same result — now with the same words.

R4 · Mirror matches and attribution

Both sides picking Sable Wren stays unambiguous: attribution on the card is her portrait and name (unchanged), the frame's colours come from SideIdentity's mirror rule (the later slot borrows a hue-distinct classic), and the rotation counter is keyed by team, not commander id, so the two Sables rotate independently.

6

Explicitly out of scope

Voice acting

Rejected in D5 for now. The seams are ready when the content is: Sfx tolerates missing streams, and the quote field is the script.

Barks at other moments

Victory and defeat lines, turn-start greetings, low-HP taunts, a "power expired" beat — a separate content feature with its own seams (power expiry is deliberately silent in the sim today; no power_deactivated event exists, and this plan doesn't add one for theatre).

A quotes toggle in Settings

The quote adds no new interruption — it rides a banner that already appears, for the same hold, and Instant already compresses it. If playtest disagrees, settings.gd's battle_animations toggle is the five-line pattern to copy.

Localization

The game ships English-only today; quotes follow the same policy as every other display string (doctrine_text, power_text). A future localization pass sweeps them together.
