Grid Commander · Battle presentation · Design of record

Battle animations — the combat cut-in

When an attack resolves, the board gives way to a full-screen battle cut-in: the two units face each other over their own terrain, squads of figures sized to their HP, the attacker's volley lands, figures fall, the counter comes back — then the map returns, exactly as Advance Wars staged it. The whole feature hangs off one method the game already awaits — BattleAnimator.animate_combat — and replays numbers the sim has already committed. It chooses nothing.

DRAFT — awaiting review 4 milestones · BA1–BA4 presentation-only + 2 core snapshot fields no new unit art

1 The seams already exist

Combat presentation is already funneled through a single awaitable: battle_animator.gd's animate_combat(result, attacker, defender), which today plays the on-map version — hit flash, camera shake, death fade. It has exactly two call sites, and both already await it, holding the interaction flow until the dust settles. The cut-in is a body swap of that one method: neither caller changes, and the AI's attacks get the animation for free because the AI issues the same commands the player does.

The one thing missing: when animate_combat runs, the units' HP is already post-combat — the command applied first. The cut-in needs to show HP falling, so CombatResult grows two snapshot fields (pre-combat displayed HP for each side), filled by the resolver at the moment it resolves. That is the entire core change, and it gets a unit test. Everything else in this plan lives in the presentation layer.

2 What plays — anatomy of the cut-in

A full-width overlay wipes in over the board. Attacker on the left, defender on the right, each posed over a strip of its own terrain with the terrain's name and defence stars on its plate. Each side shows a squad of figures sized to its HP — Advance Wars' rule: ceil(displayed_hp / 2), so 10 HP is five figures and a battered 1 HP straggler fights alone. The volley crosses, the defender's HP pips tick down to the snapshot value, surplus figures fall. If the defender survives and the result says countered, the answer comes back the other way. Deaths end in the explosion and an empty side. Then the wipe reverses and the map is back.

The beat sheet

Every beat is driven by a field of the result — the cut-in never decides an outcome, only paces it. Unskipped it runs ≈ 1.7 s for a clean kill and ≈ 2.4 s for a full exchange with counter, which is Advance Wars' tempo. Accept or cancel at any point fast-forwards every remaining beat to its end state — it never aborts, so the await always resolves exactly once (see R2).

6 · Hold + wipe out0.45 s—final tableau holds a beat; wipe reverses; the existing map aftermath (death fade, sync_sprites) runs abbreviated underneath

The call-site surgery — all of it

func animate_combat(result: CombatResolver.CombatResult, attacker: Unit, defender: Unit) -> void:
	view.refresh_sprite(attacker)
	if Settings.battle_animations and not capturing and _both_sides_visible(attacker, defender):
		await cutscene.play(result, attacker, defender)   # the cut-in; BA1
		_sync_aftermath(result, attacker, defender)       # abbreviated map beats
		return
	# animations off / capturing / fogged side: today's on-map path, unchanged
	...

_both_sides_visible asks core/rules/vision.gd — the single authority — whether the viewing team may see both combatants. Under fog an AI-versus-AI exchange the player can't see stays on the map path rather than parading a hidden unit across the screen (R6). Everything the cut-in shows, the viewer could already see.

3 Decisions

D1 — The cut-in replays a snapshot; it computes nothing

CombatResult grows attacker_hp_before and defender_hp_before (displayed HP, 1–10), filled inside CombatResolver.resolve and covered by a core unit test. The cut-in reads the result and the two units — it never touches the damage chart, the RNG, or re-derives a number. This repo has already paid for a "fourth independent opinion" bug once (movement); the animation layer will not open that door for combat.

D2 — Board art, blown up; zero new unit sprites

Squad figures are the existing 64 px PixVoxel atlas art via UnitSprite.texture_for, nearest-neighbour scaled — framed as a diorama over a terrain strip, not a fake side-view. Muzzle flashes, tracers, shells, and explosions are original procedural effects (tweened polygons / CPUParticles2D). Authoring per-unit animation sheets is the named scope-creep trap and is explicitly out (R5). Original presentation throughout — nothing lifted from Advance Wars' battle screens, per the repo's legal note.

D3 — Squad size is Advance Wars' rule

figures = ceil(displayed_hp / 2), capped at 5: full health is five figures, 1–2 HP is a lone survivor, zero only ever means dead. Figures exist for legibility — the pip bar carries the exact number, the squad carries the feel.

D4 — One persistent toggle, one skip, both presentation-only

A new tiny autoload/settings.gd persists battle_animations (default on) to user://settings.cfg — a user preference, not a match option, so it doesn't belong on MatchConfig and survives across matches. The menu gets the checkbox next to the fog toggle. In battle, accept/cancel during a cut-in fast-forwards it. Toggle off = today's on-map path, byte-identical behaviour.

D5 — Weapon styles are data, six styles cover eighteen units

How a volley looks is a BattleStyle resource under data/battle_anim/ (projectile kind, shots per figure, sfx name, timing nudges). UnitType gains battle_style: StringName — the same precedent as atlas_col, a presentation key on the data resource, no gameplay number anywhere in the style. The BattleStyle class itself lives in scenes/battle/cutscene/, keeping core/ free of presentation types.

D6 — An overlay in battle.tscn, not a scene switch; captures opt out

The cut-in is a CanvasLayer inside battle.tscn — no change_scene, the board and all state stay live underneath, and the awaitable contract is a plain child node's method. While capturing, the cut-in is suppressed exactly like the shake and pulse already are, so make screenshot stays deterministic; the scenario driver can pose the overlay directly when a screenshot of the cut-in is wanted.

4 Milestones — each one shippable

BA1

The cut-in shell — every attack cuts in, readable, skippable, optional

  • Core: attacker_hp_before / defender_hp_before on CombatResult, set in resolve; test in tests/unit/ proves the snapshots match displayed HP going in.
  • Scene: scenes/battle/cutscene/combat_cutscene.tscn/.gd + cutscene_side.gd — wipe, two posed sides (single unit portrait for now, no squads yet), name/HP-pip/terrain-star plates, white-flash impact, HP tick, counter beat, death fade, wipe out. Fast-forward on accept/cancel.
  • Wiring: the animate_combat body swap (section 2), the Settings autoload, the menu checkbox, the Vision visibility guard, capturing suppression.
  • Ships: the full loop with placeholder staging — the game feels different on day one, and turning it off restores today exactly.
BA2

Squads and volleys — fights look like fights

  • HP-sized squads (D3) in loose formation; surplus figures drop on the tick, staggered.
  • First two BattleStyle resources — small_arms and cannon — with muzzle flash, tracer/shell, per-style Sfx; every unit temporarily maps to one of the two.
  • Death explosion effect replaces the plain fade inside the cut-in; explosion.wav stays the sound.
  • Ships: infantry firefights and tank duels read at a glance; other units borrow the closest style without looking broken.
BA3

The full roster — all six styles, all eighteen units, all three domains

  • Remaining styles: autocannon (flak bursts), rocket, bomb, torpedo. Air figures hover-bob; sea figures sit on a water strip; indirect attacks (result never counters) frame the lone volley without an awkward pause.
  • Terrain strip drawn from each side's actual cell via the terrain atlas; property strips team-tinted like the board.
  • Ships: every matchup in the game stages correctly — checked against the mapping below.

BA4

Pace and polish — the feature survives a whole match

  • AI-turn pacing: consecutive AI attacks tighten the hold/wipe budgets so a five-attack turn doesn't cost twelve seconds of ceremony; skip stays instant.
  • Entrance polish — camera punch-in on the struck cell before the wipe, faction accent on the wipe itself.
  • New Sfx entries generated for the styles that want their own voice (flak, rocket, torpedo splash), added to the make sfx pipeline; missing files stay silent by design.
  • Verification soak: a full AI-vs-AI match with animations on — no softlock, no drift between cut-in pips and map HP badges, then the same match with the toggle off (see risks R1/R2). Screenshot of a posed cut-in via the scenario driver.

5 Risks

R1 · Pacing fatigue — the animation everyone turns off

Two seconds a battle is charming for ten battles and a chore for two hundred; it's why Advance Wars shipped the off switch. Mitigation is layered: fast-forward on any press, the persistent toggle, and BA4's tightened AI pacing. The BA4 soak is the gate — if watching a full AI turn grates with the mitigations, the budgets shrink before it ships.

R2 · The await contract — a cut-in that never finishes is a softlock

Both call sites hold the entire interaction flow on animate_combat. A skip racing a tween kill, or an early return path that forgets to resolve, freezes input for the rest of the session. Design answer: the cut-in owns one completion signal, skip fast-forwards state rather than aborting tweens mid-flight, and every branch funnels to the same exit. Verified by the BA4 soak plus deliberate skip-spamming at every beat.

R3 · Art readability — isometric board sprites in a close-up frame

The PixVoxel art was drawn to sit on a tile, not to fill a stage; blown up it may read as "big board piece" rather than "battle scene". BA1 exists partly to answer this cheaply: the shell ships with single portraits, and the diorama framing (terrain strip, plates, wipe) carries the drama. If a blown-up figure still reads poorly, the fallback is smaller figures and stronger effects — not new art (R5).

R4 · Capture determinism — screenshots must stay byte-stable

The repo already fought this war with the camera shake: mid-tween frames make identical captures differ everywhere. The cut-in follows the same law — suppressed while capturing, all randomness presentation-side (never game.rng), and any posed cut-in screenshot goes through the scenario driver at a settled tableau.

R5 · Scope creep toward authored animation sheets

Scope creep is this project's named top risk, and battle animation is its natural habitat — there is always one more frame to draw. D2 is the fence: board art plus procedural effects, six styles total, and any "just one bespoke sprite" proposal is a plan amendment, not a commit.

R6 · Fog and hidden units — the cut-in must not out-see the viewer

Handled at the gate: the cut-in only plays when core/rules/vision.gd says the viewing team may see both combatants; otherwise the exchange stays on the map path, which already renders fogged units correctly. No new visibility logic anywhere — the single authority stays single.

6 Explicitly out of scope

  • Authored side-view animation sheets — the whole of R5. Revisit only as its own plan with an art budget.
  • Per-team animation toggles (Advance Wars' "own / enemy / off" matrix) — one switch plus skip covers the need; add granularity only if playtesting demands it.
  • Commander cut-in portraits during battle — the power banner already owns commander presentation; mixing it into combat beats is a separate design.
  • Capture / build / resupply cutscenes — this plan is combat only; the on-map language for everything else stays.
  • A replay theater — the command log makes one possible someday; it is not this.

7 Acceptance checks

  • Every resolved attack — player or AI — plays the cut-in when animations are on and both sides are visible to the viewer; the toggle off reproduces today's behaviour exactly.
  • Cut-in HP pips, falling figures, and survivor counts always equal the sim's snapshot numbers; the map's HP badges agree the moment the wipe clears. No number is computed in the presentation layer.
  • Accept/cancel at any beat lands on the correct final tableau and returns control immediately; skip-spamming through a full AI turn never softlocks.
  • make test stays green, including the new CombatResult snapshot test; make screenshot output is unchanged with the feature merged.
  • All eighteen units stage with the correct style, terrain strip, and stars, across land, air, and sea matchups — including indirect fire and an attack on a visible submarine.
  • No file under core/ or ai/ references a Node, the cut-in scene, or a style resource; battle_style on UnitType is a plain StringName.
