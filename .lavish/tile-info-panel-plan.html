Grid Commander · UI plan · 2026-07-20

Make the unit the headline of the tile panel

When the cursor sits on an occupied tile, the corner panel still leads with terrain and crams the unit into one dash-separated line. Flip the hierarchy: unit card on top with readable current/max stats, terrain demoted to a compact card below — the classic AW layout.

Scope

Presentation-only. Three files, ~120 lines, one afternoon.

No core/ or ai/ changes, no new GUT tests (per testing policy), fog & doctrine hiding preserved.

03

Implementation

Three steps, in order. The two show_* methods merge into one show_tile() because terrain rendering now depends on whether a unit is present (which move classes to list). Only battle_view.gd:322 calls the panel, so the API break is contained.

Step 1 — scenes/ui/terrain_panel.gd: merge into show_tile()

Unit card first (icon + name + team, HP x/10, Fuel x/max · Ammo x/max, extras line), then the terrain card with its tile icon. Defense as stars, move costs filtered to the occupant's class. The unit icon reuses UnitSprite.texture_for() — the static helper that exists precisely so menus can show board artwork; the terrain icon is one cell of the terrain atlas, team-tinted row for owned properties (the 64px cell size is duplicated as a const rather than reaching into BattleView). The diff below is the complete old → new file.

Step 2 — scenes/battle/battle.tscn: rebuild the panel's rows (in the editor)

The fog/doctrine gate (can_see_unit, added in the Commanders merge) stays exactly where it is — the panel keeps receiving null for units the viewer must not see.

05

Risks & verification

Layout

First-frame clipping

battle.tscn pins the panel with fixed offsets (80 px tall); the unit card can double its height. set_side() already re-applies a min-size preset on every cursor move — additionally call it once in _ready() so the very first frame is sized right.

Font

Star glyphs

DEF stars assume ★/☆ exist in Godot's default UI font. Verify in-game on first run; the fallback is the plain-number option in Decisions, a one-line change.

Preserved

Fog & doctrines

Hidden units never reach the panel — refresh_panel nulls anything failing can_see_unit() (fog and Vanish-style doctrine hiding, via the Vision authority). The redesign doesn't touch that gate.
